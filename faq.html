Github collaboration

Pingendo's collaboration is built on top of Github. 
Github knowledges are required to set up a collaboration project.
For contributors, Github is behind an easy interface that does not require any knowledge of Github.

Editor

The editor features dev and non-dev features. Users can choose to remain on the surface or to go deep according to everyone's role and skill. The WYSIWYG editor permits to all contributors to perceive real-time the effect of their modification. Some use cases:

  • Copywriters edit texts.
  • Photographers update links to photos.
  • Designers edit lay-out or styles.
  • SEO specialists add page title or description, revise texts and more... 
  • Marketers act where he/she can and distribute tasks.
  • Stakeholders can be main contributors or just be set "read only" and approve the job at required stages.
  • Developers focus on issues the above people can't. e.g. effects, custom responsive rules, deep styling...

2) What is a "Pull request"

It's a copy of the original page (master) in order to be:

  • Edited
  • Closed (discarded)
  • Merged (accepted) by the PM                                         
  • Closed (discarded) by the PM 
  • Compared to the master in every single file.

3) How to create pull requests efficiently?

To help the project manager to revise your pull-requests, please remind to:

  • Give them a title related to the changes applied                   
  • Create a pull request for each different set of changes
  • Keep them small, with few changes each. It helps the PM to quickly compare and understand the proposed changes
  • Each pull request will be part of the powerful history of Github. Keeping them small means a granular "time machine" at your fingertips
  • Comment on each pull request for clarity to the whole team.

4) How to manage contributors efficiently?

To help contributors to create in the required direction please remind to:

  • Revise pull requests often
  • Comment

7) Do users have to signup to Github to submit pull requests?

Yes, because Github manages users and their actions. Signing up to Github does not require any dev skill, It's a plain signup form.

    8) Does Pingendo save developer's time?

    Yes, we would like to. The Github integration permits to developers to manage a "safe environement" where users can iterate on content or source, each one at their own skill level.
    Our goal is to keep developers at fingertips of the marketing and communication team for technical issues only. Easy to reach, with clear requests related to real pages. 

    9) Does Pingendo save time to marketing and communication teams?

    We hope to. Pingendo is really effective on prototyping or when iteration is required on the whole page elements. When pages are too fluid for a traditional CMS.
    Screenshots, mockups, excel files with data, presentation, Word documents with texts and other means useful to explain things to developers or stakeholders, are not required any more.
    Iteration is taken straight to real pages, where it should be to make things simpler.

    9) How to invite contributors to public repositories?

    Send them the project link. The contributor has to login on Github to submit pull request.

    9) How to invite contributors to private repositories on personal plan?

    • Ask contributors to signup to Github
    • Set them as collaborators
    • Send them the link to the project 

    9) What changes for Pingendo with private or public repositories or with personal or organization plan?

    Non of them makes any difference for Pingendo. Privacy settings and user settings are managed by Github only.

    9) I'm a developer and I have many private repositories. How to grant access to Pingendo only to the project related repositories?

    Github API does not yet permit a granular access to repositories.
    A developer can adopt an alternative method by putting a new account between Pingendo and your all repos. It also helps to keep sharing clean and simple.

    • Create a Github user e.g. "Webproject company abc" 
    • Grant access to that user to the repositories you would like to share
    • Grant access to Pingendo to the user "Webproject company abc".

    9) Is granting access to my repositories to Pingendo secure?

    Pingendo does not store any Github password. Not a single byte of your source is sent to Pingendo. Pingendo has no servers out of one to download the app and one for billing. 
    When you grant access to your repositories, your browser permits to Pingendo to talk to Github.

    9) Who can access the repositories activated on Pingendo?

    Roles and permission are managed by Github settings.
    Pingendo just read and write your page on the activated repositories. 

    Who can read source on Github?

    • Public repos --> Everybody
    • Personal plans, private repos --> Owners and Collaborators
    • Organization plans, private repos --> Read permission and up

    Who can read pages rendered by Pingendo?

    • Everybody who can read your source and has the pagelink created by Pingendo 

    Who can submit pull-requests by Pingendo?

    • Everybody who can read your source and has an account on Github.

    Who can merge pull-requests?

    • Personal plans --> Owners and Collaborators
    • Organization plans --> Users with write permission and up
