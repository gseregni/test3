Looking for the desktop Bootstrap builder?

Efficient workgroup iteration since first prototype. On Github flow.

Developers, content creators, designers or stakeholders contributing on the same HTML/CSS source file. 
Each one according to his/her own role and skill. 
Everybody safely managed by developers on Github flow. More...

1 - Developer
Setting up Github

Developer creates the repository for the web pages on Github and set permissions.

Github is where developers start when adding features to webpages.
Taking non-dev contribution there, saves tons of time to developers and speeds up collaboration.

2 - Users
Making their own copy

Users create their own pull request.

A pull request is a copy of the master where users can freely iterate. 
A simple interface hides Github complexity to non-developers. 

3 - Users
Making changes

Users modify the pull request by the simple web editor. 

Each one according to its own role and skill.
Copywriter modifies texts, designer act on colors or the UI, developer add animations or whatever required. 

4 - Developer or Project owner
Accepting or denying changes

Developer and the project owner accept or deny the pull requests.

Back to point 2 for next iteration.
