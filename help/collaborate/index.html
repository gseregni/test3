Make teams collaborate.

Learn how to make developers, creatives, content-makers and stakeholder work together.

Share preview

Pingendo includes an handy service to preview your pages.
To share the link for preview with others click the  button from the builder toolbar, or from the project dashboard.

Sharing direct link works with everyone with public projects. To preview pages in privare repos users need to be collaborators.

Invite collaborators

To start collaborate with others you should first add them to your project. 
From the project page click on the button to open its collaborators, type the Github username for the new collaborator and click the "Add" button.

Propose changes

Collaborators can propose changes as "Pull request".  They will be accepted or discarded by the project owner. To create a new pull request click on the   button.

A new pull request makes a copy of the entire site, so you can safely make your modifications. 

Accept changes

Project owners and collaborators can merge changes contained in a pull-request.
If the pull-request is mergeable, you can click on the 'Merge' button when the pull-request is selected.

Otherwise, if the pull-request contains conflict a manual merge will be required. Click the 'Merge manually' button to  navigate to Github and merge manually conflicting files.

Resolve files conflits and marge manually pull-requests requires good HTML and CSS knowledge.

Collaborate from desktop

If you are a user of Desktop version of Pingendo, to collaborate with your team working on Pingendo web, you are required to use Github Desktop client to create pull-requests and commit changes.
