Introduction

Pingendo is a Bootstrap builder available in three versions:  
Pingendo Playground.  It's a free Bootstrap playground for the community, 
Pingendo Desktop. It works locally on filesystem  and offline,
Pingendo for Github. It works online integrated with Github 


Pingendo Playground

Pingendo Playground it's a web Bootstrap HTML and Saas playground, available here  playground.pingendo.com
You can create, save and share your creations with others for free.
HTML and SASS will be saved as Gists on Github.

Pingendo Desktop

Pingendo Desktop works on Windows (7+), OSX (10.9+) and Linux.  
You can get a copy at www.pingendo.com/desktop.html  

Once installed and launched you can create a page from a template. 
The result is saved as plain HTML, Sass and CSS files that you can modify also with external editors. No need to export source.
To learn more about what you can do in Pingendo jump to the Editor section .

Pingendo for Github

Pingendo for Github is a web app that works in Chrome or Safari. Support for other browsers will be added.
It integrates with Github API to load and save HTML pages and theirs assets in repositories.
When you access Pingendo, you'll be required to grant Pingendo access to Github API. That's in order to let PIngendo read and save files stored in the repository.  
We use standard OAuth mechanism provided by Github and we do not store your password in any way. 
For additional info check out the Github website https://help.github.com/articles/connecting-with-third-party-applications/


Builder

The builder interface is the same for all three versions and consists of 5 main areas 

1 - Toolbar 

2 - Sections and Components panels 
Here you can browse and drag pre-made sections and Bootstrap 4 components

3 - Canvas
You can drop elements here and select to edit text or properties                                      
4 - Page and Selection panels
Set properties here                                     

5 - HTML and Saas source panel
Edit the source live.   
                                                                                                                                              




HTML

Pingendo lets you build good-looking and perfectly coded HTML pages, deadly fast. 

To start mocking up a new page you can drag needed elements from the Sections and Components panel.
Pre-made sections are those commonly used in site development.  
Components are all the ones available on Bootstrap 4.              

Click to select elements on canvas and play with its properties in the Selection panel.   
Properties are the ones offered by Bootstrap.

You can select element's parents by the breadcrumb.
This is particularly useful when you can't pick an element directly from canvas.
The ESC key moves your selection one parent up.                  

If you are working on something more than a quick mockup, you can get your hands dirty by using the editor in the bottom HTML tab.

The editor is live, so the canvas will update almost realtime to reflect changes and vice versa. 
Since what you are editing it's a live HTML DOM, you should  use Pingendo only on proper and valid HTML  documents. 
Preprocessors (like PHP) or templates engines are not supported and can't be saved correctly.


Saas

Saas is the CSS preprocessor used by Bootstrap 4.  
Unlike most Boostrap-based editors, Pingendo let you customize directly the Saas source and not simply the CSS.
That's in order to generate clean sources easily readable by developers.

You can play with most common Saas variables,  such as color schemes and fonts, by the page panel.  If you need to customize more  you can edit the Sass source from the Sass tab bottom.


When you create a new page the default Boostrap 4 theme will be attached.
As soon as you modify something you will be prompted to name and save it.

To check which is the SASS theme attached to a page, look at the FILE property of the page tab.

To attach an existing theme to a page click on the "FOLDER" icon of the file row and select the desired theme.   Coming

Github integration

This part applies only to Pingendo for Github users. Pingendo Desktop users should  use Github Desktop client instead.

Pingendo for Github features Github integration to let you work online on pages stored in Github repositories.
Scope of the integration is to make simple for teams of developers and non developers to efficiently collaborate building static web sites.
To make this easy for non-devs, Pingendo for Github features an alternative and simplified version of the Github web interface.

Pingendo for Github uses the same url scheme of Github. 
If, for example you need to access the repository test of exampleuser, through Pingendo, just point your browser to github.pingendo.com/exampleuser/test. You'll reach the Github repository github.com/exampleuser/test.

Permissions

Repository permissions are configured on Github. 

- Collaborators
They can edit the master version as well merge and close pull request.
                             
- Any Github user
They can submit new pull-requests only. No permissions to merge or edit anything in the repository.

Sharing

- Share URL
You can share the URL to any Pingendo page as repository page or single files. User can perform actions according to their permissions. 

- Share preview URL 
This is a view only link rendered in any browser. It's useful to share the online page with non-Github users or to test it on different devices. It's served by .... so cannot be used for production, to avoid xxxx github

Pull requests

(Pull request can be called "Propose change" or "Propose new version" to help non techie to better understand what it is.)
Pull requests let you tell others about changes you want to push to the master version. 
Once a pull request is sent, interested parties can review the set of changes, discuss potential modifications, and even push follow-up commits (versions) if necessary.

To create a new pull request click the "New pull request" button and give it a name that well describe what your changes are about.
Once created you'll get a fresh copy of the master version to work with safely.

As soon as you modify or delete existing files or upload a new one, a status badge will appear beside the file name. 

To discard a pull request use the Close command. 
If a pull request is okay and you are a collaborator of the  project, you can merge it to the master version.

Warning, Git checks conflicts, so Pull requests may not be sometime merge-able.
This happens when a pull request modifies a part of a file that has also been modified in the master version.
In these cases, a manual merge by developer will be necessary.

 

Github Pages

Github Pages is a free hosting service kindly offered by Github. 
The master version of every repository can be published to Github pages and made available on the internet.

To publish the master version use the Publish button. Once published you can visit the web site by clicking "Visit site" or by pointing your browser to <username>.github.io/<project>.
