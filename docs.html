Introduction

Pingendo is a Bootstrap HTML and Saas builder available in three versions:  
Pingendo Playground. Online Bootstrap playground for community and education.  Build and share pages and snippets.
Pingendo Desktop. Desktop builder for developers. It works locally and offline to integrate into the developer's toolset. 
Pingendo for Github. Online builder for teams and clients integrated with Github.  Lets non-devs work on Github flow.


Pingendo Playground

Button

Pingendo Playground is a web Bootstrap HTML and Saas playground, available here  playground.pingendo.com.
You can create, save, and share your creations with others for free.
Every page will be saved as a Gist on Github.

Pingendo Desktop

Pingendo Desktop works on Windows (7+), OSX (10.9+) and Linux.  
You can get a copy at www.pingendo.com/desktop.html.                               

Once installed and launched, you can create a page from a template. 
The result is saved as plain HTML, Sass, and CSS files that you can also modify with external editors. No need to export source.
To learn more about what you can do in Pingendo, jump to the Builder section .

Pingendo for Github

Pingendo for Github is a web app that works in Chrome or Safari. Support for other browsers will be added.
It integrates with Github API to load and save HTML pages and their assets in repositories.
When you access Pingendo, you'll be required to grant Pingendo access to Github API. That's in order to let Pingendo read and save files stored in the repository.  
We use standard OAuth mechanism provided by Github and we do not store your password in any way. 
For additional info check out the Github website https://help.github.com/articles/connecting-with-third-party-applications/.


Builder

The builder interface is the same for all three versions and consists of 5 main areas: 

1 - Toolbar 

2 - Sections and Components panels 
Here you can browse and drag pre-made sections and Bootstrap 4 components.                                     

3 - Canvas
You can drop elements here and make selection to edit text or properties.                                             
4 - Page and Selection panels
Set Bootstrap properties here.                                                                                                           

5 - Source panel                                     
Edit HTML and Saas source.   
                                                                                                                                              




HTML

Pingendo lets you build good-looking and perfectly coded HTML pages, deadly fast. 

To start mocking up a new page you can drag needed elements from the "SECTIONS" and "COMPONENTS" panels.
Pre-made sections are those commonly used in site development.  
Components are all those available on Bootstrap 4.              

Click to select an element on canvas and play with its properties in the "SELECTION" panel.   
Properties are those offered by Bootstrap.

You can select element's parents by the breadcrumb.
This is particularly useful when you can't pick an element directly from canvas.
The ESC key moves your selection one parent up.   

To set up the document title and SEO related properties use the "PAGE" panel.


Saas

Saas is the CSS preprocessor used by Bootstrap 4.  
Unlike most Boostrap-based editors, Pingendo let you directly customize the Saas source and not just the CSS.
That's in order to generate clean sources that are easily readable by developers.

You can play with most common Saas variables,  such as color schemes and fonts, by the page panel.  If you need to customize more you can edit the Sass source from the Sass tab at the bottom.


When you create a new page the default Boostrap 4 theme will be attached.
As soon as you modify something you will be prompted to name and save it.

To check which SASS theme is attached to a page, look at the FILE property of the page tab.


Sources

If you are working on something more than a quick mockup, you can get your hands dirty by using the HTML and Sass editor in the bottom tab.

The editor is live, so the canvas will update realtime to reflect changes and vice versa. 

Pingendo works only with proper and valid  HTML documents.  Preprocessors (like PHP) or HTML fragments (like Angular templates) are not supported and won't be saved correctly.

Github integration

This part applies only to Pingendo for Github users. Pingendo Desktop users should  use Github Desktop client instead.

Pingendo for Github features Github integration to let you work online on pages stored in Github repositories.
The scope of the integration makes it simple for teams of developers and non developers to efficiently collaborate building static web sites.
To make this easy for non-devs, Pingendo for Github features an alternative and simplified version of the Github web interface.

Pingendo for Github uses the same url scheme of Github. 
If, for example, you need to access the repository test of exampleuser , just point your browser to github.pingendo.com/exampleuser/test  in Pingendo. You'll reach the Github repository github.com/exampleuser/test. 


Account page

From the account page you can browse your existing Github repositories.
Repositories are divided in two groups, the ones you own and the ones you have contributed to.

You can create a new repository using the "Create new repo" command.


Repository page

The repository page is divided into three columns:

1 - Repository browser
This shows collaborators and allows collaborators to browse and select versions.
                                                                                                                      
2 - Files browser
- select pages and assets contained in the current version
- view status changes on files

3 - Preview
- open file in the builder
- open file in the browser
- make actions on file


Permissions

Repository permissions are configured on Github. 
Collaborators of the repository can edit the master version as well merge and close pull request.
Other Github users that are not collaborators can only submit new pull requests. 


Sharing

To share pages with non-Pingendo users, open the page in "Preview" and share its link. 
Note that files served by git.pingendo.com are intended for low-traffic testing or for sharing temporary demos with a few people during development.
This is NOT a CDN or a hosting service.
If you are looking for a hosting solution look at the " Github Pages" section.



Pull requests

(Pull request can be called "Propose change" or "Propose new version" to help non-devs to better understand what it is.)
Pull requests let you tell others about changes you want to push to the master version. 
Once a pull request is sent, interested parties can review the set of changes, discuss potential modifications, and even push follow-up commits if necessary.

To create a new pull request click the "New pull request" button and give it a name that describes what your changes are about.
Once created you'll get a fresh copy of the master version to work with safely.

As soon as you modify or delete existing files or upload a new one, a status badge will appear beside the file name. 

To discard a pull request use the "Close" command. 
If a pull request is okay and you are a collaborator of the  project, you can merge it into the master version.

Warning, Git checks conflicts, so pull requests may not be sometime merge-able.
This happens when a pull request modifies a part of a file that has also been modified in the master version.
In these cases, a manual merge by developer will be necessary.

 

Github Pages

Github Pages is a free hosting service kindly offered by Github. 
The master version of every repository can be published to Github pages and made available on the internet.

To publish the master version use the Publish button. Once published you can visit the web site by clicking "Visit site" or by pointing your browser to <username>.github.io/<project>.
